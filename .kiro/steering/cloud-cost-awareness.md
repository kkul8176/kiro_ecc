# 클라우드 비용 의식 원칙

> 이 규칙은 클라우드 자원을 생성/변경/설계하는 모든 작업에 적용된다.

## 핵심 원칙

1. **비용 영향 사전 안내**: 자원을 생성하기 전에 예상 비용을 사용자에게 안내한다.
2. **환경별 사양 차등**: 개발/테스트 환경은 최소 사양을 사용한다.
3. **불필요 자원 정리**: 사용하지 않는 자원은 즉시 정리하고, 정리 방법을 문서화한다.
4. **비용 태그 필수**: 모든 자원에 비용 추적용 태그를 적용한다.

## 환경별 사양 가이드

| 환경 | 원칙 | 예시 |
|------|------|------|
| dev | 최소 사양, 필요할 때만 실행 | t3.micro, Fargate 0.25vCPU, 단일 AZ |
| stg | 프로덕션과 유사하되 축소 | t3.small, Fargate 0.5vCPU, 단일 AZ |
| prd | 요구사항에 맞는 적정 사양 | 성능 테스트 기반 결정, 멀티 AZ |

## 비용 최적화 체크리스트

- [ ] 개발 환경에 프로덕션 사양을 사용하고 있지 않은가
- [ ] 사용하지 않는 EBS 볼륨, 탄력적 IP, 로드밸런서가 없는가
- [ ] NAT Gateway가 꼭 필요한가 (개발 환경에서는 VPC 엔드포인트로 대체 가능)
- [ ] RDS/Aurora가 개발 환경에서 멀티 AZ로 설정되어 있지 않은가
- [ ] CloudWatch 로그 보존 기간이 적절한가 (개발: 7일, 프로덕션: 90일)
- [ ] 예상 월간 비용을 산출하여 문서화했는가
