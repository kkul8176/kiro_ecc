# UI/UX 디자인 원칙

> 이 규칙은 UI가 포함된 모든 작업에 적용된다. Kiro가 직접 프론트엔드를 구현할 때, code-reviewer가 UI 코드를 리뷰할 때, designer 에이전트가 디자인할 때 모두 참조한다.
> 코드 수준의 프론트엔드 패턴(React 컴포넌트, 훅, 상태관리 등)은 `skills/frontend-patterns`를 참조한다.

## 네비게이션 흐름

- 사용자가 목적지에 **최소 클릭**으로 도달할 수 있도록 설계한다.
- 주요 기능은 1~2클릭 이내에 접근 가능해야 한다.
- 현재 위치를 항상 알 수 있어야 한다 (브레드크럼, 활성 메뉴 표시 등).
- 뒤로 가기, 홈으로 돌아가기가 항상 가능해야 한다.

## 정보 구조 (Information Architecture)

- 관련 콘텐츠를 논리적으로 그룹핑한다.
- 중요도에 따라 우선순위 배치한다 (가장 중요한 정보가 가장 먼저 보임).
- 한 화면에 너무 많은 정보를 넣지 않는다 — 점진적 공개(progressive disclosure) 활용.
- 일관된 레이블링과 용어를 사용한다.

## 시각적 계층 구조

- **타이포그래피 스케일**: 제목 → 부제목 → 본문 → 캡션의 크기/굵기 차이가 명확해야 한다.
- **색상 대비**: 텍스트와 배경의 대비율 최소 4.5:1 (WCAG AA 기준).
- **여백(Spacing)**: 관련 요소는 가깝게, 무관한 요소는 멀리 배치 (근접성 원칙).
- **시선 유도**: 핵심 CTA(Call to Action)가 시각적으로 가장 눈에 띄어야 한다.

## 반응형 디자인

- 모바일 퍼스트로 설계하고, 데스크톱으로 확장한다.
- 브레이크포인트: 모바일(~768px), 태블릿(769~1024px), 데스크톱(1025px~).
- 터치 타겟 최소 44x44px (모바일).
- 이미지와 미디어는 뷰포트에 맞게 조절한다.

## 엣지 케이스 UI (필수)

모든 데이터 표시 영역에 다음 상태를 고려한다:

| 상태 | 처리 |
|------|------|
| 로딩 중 | 스켈레톤 UI 또는 스피너 표시 |
| 빈 상태 | 안내 메시지 + 다음 행동 유도 (예: "아직 항목이 없습니다. 추가해보세요") |
| 에러 상태 | 사용자 친화적 에러 메시지 + 재시도 버튼 |
| 부분 로딩 | 로드된 부분은 표시, 나머지는 로딩 표시 |
| 긴 텍스트 | 말줄임(...) 또는 더보기 처리 |

## 인터랙션 패턴

- **피드백**: 모든 사용자 액션에 즉각적 피드백 (버튼 클릭 → 시각적 변화, 폼 제출 → 로딩 표시).
- **트랜지션**: 상태 변화 시 부드러운 전환 (150~300ms). 갑작스러운 레이아웃 점프 금지.
- **호버/포커스**: 인터랙티브 요소에 호버/포커스 상태 표시.
- **비활성화**: 사용 불가능한 액션은 비활성화 상태로 표시하고, 이유를 툴팁으로 안내.

## 접근성 (Accessibility)

- 모든 이미지에 `alt` 텍스트 제공.
- 폼 요소에 `label` 연결.
- 키보드만으로 모든 기능 사용 가능.
- 포커스 순서가 논리적.
- 색상만으로 정보를 전달하지 않는다 (색맹 고려).
- ARIA 속성을 적절히 사용 (`role`, `aria-label`, `aria-expanded` 등).

## 디자인 시스템 기본 요소

프로젝트에 디자인 시스템을 구성할 때 최소한 다음을 정의한다:

- **색상 팔레트**: Primary, Secondary, Accent, Background, Text, Error, Success, Warning
- **타이포그래피**: 폰트 패밀리, 크기 스케일 (h1~h6, body, caption)
- **간격(Spacing)**: 4px 또는 8px 기반 그리드
- **모서리 반경(Border Radius)**: 일관된 라운딩 값
- **그림자(Shadow)**: 깊이 표현을 위한 그림자 단계

## 유사 서비스 조사 (디자인 작업 시)

본격적인 디자인 작업 전에 다음을 수행한다:
- 유사 서비스/경쟁 제품의 디자인을 웹 검색으로 조사
- 업계 디자인 트렌드 파악
- Best Practice 사례 수집
- 조사 결과를 디자인 결정의 근거로 활용
